## 1. 목표와 우선순위

| 순위 | 목표 | 의미 |
|------|------|------|
| **1순위** | 포트폴리오 / 학습 | Kafka·Docker·이벤트 파이프라인 등 기술 스택 경험 + 코드 소유권 확보 |
| 2순위 | 실수익 | 북극성 목표(스트레치): **실거래 시 월 +20%** (예: 10만원 → 12만원) |

**우선순위의 귀결:**
- Kafka·12서비스 유지 (다이어트 안 함 — 학습 가치)
- 수익보다 이해(학습노트)·테스트·문서가 우선
- 수익 지표는 "기록"이지 "판정 기준"이 아님 (게이트 분리, §2)

## 2. 게이트 (Go/No-Go 판정 기준)

> 목표(꿈)와 게이트(통과 조건)는 다르다. 게이트를 통과해야 다음 단계로 간다.

| 게이트 | 조건 | 통과 시 |
|--------|------|---------|
| **G1. 테스트넷 1달** | ① 신호→알림→주문→가드 전 과정 **무사고** (주문 누락·오발동·가드 오작동 0건) ② **MDD ≤ 15%** ③ 수익은 기록만 | 실거래 진입 |
| **G2. 실거래 10만원 1달** | ① MDD ≤ 15% (하드) ② 월 +20%는 목표(스트레치) ③ 위반 시 **중단 + 회고** | 증액 검토 |

근거: 10만원 스케일에선 GPT API비(월 $5~15)가 목표수익(2만원)과 맞먹음 → 첫 달의 본질은 돈이 아니라 **기계 검증**.

## 3. 핵심 가설과 검증 계획

**가설:** 특정 유튜버의 지지/저항/파동 분석에 엣지가 있다.
- 근거: 장기 직접 관찰 (도메인 지식 — 단, 기억은 적중 편향 가능)
- 상태: **정량 미검증** — 백테스트에서 유튜버 신호 진입 0건 (번역 문제로 측정 자체가 안 됨)

**인지된 핵심 문제 — "번역 손실":** 사용자의 시선이 코드로 옮겨지지 않음.
- 증거: 주봉 박스 추출되고도 `is_reference_only`로 폐기 / `absolute_stop` 자주 NULL / 시뮬 진입 0건
- 해법: **신호엔진 재설계** (방법론: 실사례 차트 + 사용자 판단 → 규칙 번역 → 백테스트 검증 → 코드)
- 요구사항 전달 규칙: 추상 요구 대신 **"실제 사례 1개 + 기대 동작"**

**검증 계획:** 백필 2년치 + `backtest.runner`로 유튜버 신호 표본 축적 → 엣지를 숫자로 확정 (재설계 전까지)

## 4. 리스크 대장

| 리스크 | 등급 | 상태 / 완화 |
|--------|------|-------------|
| 유튜버 의존 (중단·변심·스타일 변화) | 🟠 | **인지·수용** (장기 관찰 신뢰). 백업 신호원 = `/market` 자체 분석 |
| 계정/멤버십 차단 (약관 위반 소지) | 🟠 | 법적 리스크보다 **운영 리스크** (차단 = 시스템 사망). 보수적 크롤링 주기 유지, 콘텐츠 재배포 금지 |
| 청산빔 (레버리지 꼬리 리스크) | 🔴→✅ | **해결됨** — 청산 가드 (BTC≤5x/ETH≤3x + 청산가 vs 최저지지 차단) |
| 시스템 장애 (컨테이너 사망 미인지) | 🟠 | 크롤러 25시간 사망 전례. **헬스체크 알림 백로그 상위** |
| API 비용 | 🟡 | 소액 스케일에선 수익 초과 — G2까지는 학습비로 수용 |
| 포폴 공개 시 인상 | 🟡 | 유튜버 **익명화**, 개인 연구용 명시 |

## 5. 범위 (v1)

**IN:** 주봉 시나리오 프레임 신호엔진(재설계) · 청산 가드 · 텔레그램 알림/명령 · 백테스트 도구 · USDT.D 매크로 게이트(재설계에 포함, 데이터소스 교체 필요)
**OUT(백로그):** SEMI_AUTO 승인 버튼 · 뉴스-포지션 연동 고도화 · 멀티유저 · 게시글 수정 감지

## 6. 운영 / 배포

- **배포 목표 변경: ~~AWS t2.micro~~ → Oracle Cloud 무료 ARM (4 OCPU / 24GB)**
  - 근거: 스택 현실 최소 RAM 3~5GB, 7.5GB에서도 OOM 실측. t2.micro(1GB)는 물리적 불가
  - 이미 착수 흔적: `create_oci_instance.py` (주의: ARM 호환, 서울 리전 용량 재시도)
- 로컬 임시 운영: Docker Desktop 메모리 12GB+ 증설
- 시스템 헬스 알림(컨테이너 다운 감지) 추가 — 백로그 상위

